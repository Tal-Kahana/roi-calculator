## מה הפרויקט עושה

מאפשר לבעלי עסקים ומשווקים להזין שלושה נתונים ולקבל ניתוח מלא:

| קלט | הסבר |
|---|---|
| הכנסות חודשיות נוכחיות | מצב ההכנסות היום |
| תקציב שיווק חודשי | כמה משקיעים בפרסום |
| אחוז צמיחה שנתי משוער | הצמיחה הצפויה בזכות השיווק (5%–30% סביר) |

**תוצאות מוצגות:**
- הכנסות חזויות לאחר 3 / 6 / 12 חודשים
- הכנסות שנתיות חזויות
- תקציב שיווק שנתי
- רווח / הפסד כולל
- ROI שיווקי באחוזים
- גרף קווי של עליית ההכנסות לאורך 12 חודשים
- המלצת AI מ-GPT-4o-mini

## נוסחאות החישוב

### הכנסות חזויות לאחר N חודשים
```
הכנסות(N) = הכנסות נוכחיות × (1 + צמיחה שנתית / 100) ^ (N / 12)
```
> הצמיחה היא **שנתית** — לכן מחלקים ב-12 לקבלת הכוח החודשי (ריבית דריבית).

**דוגמה:** הכנסות ₪50,000, צמיחה 24% שנתית, אחרי 6 חודשים:
```
50,000 × (1.24) ^ (6/12) = 50,000 × 1.113 = ₪55,678
```

### ROI שיווקי שנתי
```
ROI = ((הכנסות שנתיות חזויות − הכנסות שנתיות נוכחיות − תקציב שיווק שנתי)
       / תקציב שיווק שנתי) × 100
```

- **הכנסות שנתיות נוכחיות** = הכנסות חודשיות × 12
- **הכנסות שנתיות חזויות** = הכנסות לאחר 12 חודשים × 12
- **תקציב שיווק שנתי** = תקציב חודשי × 12

ROI חיובי = ההשקעה משתלמת | ROI שלילי = ההשקעה לא מכסה את עצמה

## פריסה ב-Vercel

הפרויקט פרוס ב-Vercel עם Serverless Function שמגינה על ה-API Key.

**למה Vercel ולא GitHub Pages?**
GitHub Pages מגיש קבצים סטטיים בלבד — לא יכול להריץ קוד שרת. Vercel תומך גם בקבצים סטטיים וגם בפונקציות שרת, לכן ה-API Key נשמר בצד השרת ולא נחשף בקוד.

**תהליך הפריסה:**
1. Push ל-GitHub → Vercel מזהה אוטומטית ומפרס
2. API Key מוגדר תחת **Settings → Environment Variables** ב-Vercel
3. הפרונטאנד קורא ל-`/api/recommend` (לא ישירות ל-OpenAI)
4. הפונקציה ב-`api/recommend.js` קוראת ל-OpenAI עם המפתח מ-`process.env.OPENAI_API_KEY`

**עדכון הפרויקט:**
```bash
git add .
git commit -m "תיאור השינוי"
git push
# Vercel מפרס אוטומטית תוך ~30 שניות
```

## חיבור ל-AI

- **מודל:** GPT-4o-mini (מהיר וזול)
- **Prompt:** שולח את כל נתוני החישוב ומבקש ניתוח בעברית ב-3–5 משפטים
- **תשובה:** ניתוח + המלצה ברורה (כדאי / לא כדאי / כדאי בתנאים מסוימים)

**להחלפת ה-API Key:**
נכנסים ל-Vercel → הפרויקט → Settings → Environment Variables → עורכים את `OPENAI_API_KEY`

> ⚠️ לעולם אל תכניסו API Key ישירות ל-HTML/JS — GitHub סורק ומזהה מפתחות חשופים וחוסם את ה-push.
